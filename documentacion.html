Introducción

El sistema M.A.I.R.A. (Mesa de Arena Interactiva de Realidad Aumentada) es una herramienta avanzada diseñada para mejorar el proceso de planeamiento militar y la toma de decisiones estratégicas en tiempo real. Su principal función es permitir a los usuarios la creación, gestión y visualización de calcos interactivos sobre mapas utilizando simbología militar estandarizada. Esto facilita la planificación de operaciones, el análisis de terrenos y la coordinación entre múltiples equipos.

Modalidades

Modo Planeamiento

En esta modalidad, el usuario tiene control total sobre los símbolos y los elementos en el mapa. Puede mover libremente los símbolos, dibujar Medidas de Coordinación y Control (MCC), y crear escenarios detallados para la planificación operativa.

Modo Juego de Guerra

Aquí, el sistema se basa en turnos y simula condiciones reales de terreno, clima y otros factores que afectan el desplazamiento y el combate de las unidades. Los elementos no se mueven libremente por el mapa; en su lugar, se desplazan según órdenes impartidas a cada unidad.

Actualmente, MAIRA está en fase de desarrollo y se encuentra en la versión 1.4.1. Todos los aportes y comentarios de los usuarios son bienvenidos para mejorar continuamente el sistema y adaptarlo a nuevas necesidades operativas.

Tecnologías y Bibliotecas Utilizadas

Mapa Base

  • Leaflet.js (v1.7.1): Biblioteca principal para la representación y manejo de mapas interactivos.
  • OpenStreetMap: Fuente de datos cartográficos base.
  • Thunderforest: Proveedor de capas de mapa adicionales (Landscape, Transport, Outdoors).

Simbología Militar

  • milsymbol.js (v2.0.0): Biblioteca para la representación de símbolos militares según el estándar MIL-STD-2525.

Sistemas de Coordenadas y Proyecciones

  • Proj4.js (v2.7.5): Biblioteca para transformaciones entre sistemas de coordenadas.
  • Proj4Leaflet: Integración de Proj4.js con Leaflet para soporte de proyecciones personalizadas.

Herramientas de Medición y Análisis

  • Leaflet.Elevation: Plugin para generar perfiles de elevación.
  • turf.js: Biblioteca para análisis geoespacial y cálculos.

Interfaz de Usuario

  • Font Awesome (v5.15.3): Iconos vectoriales utilizados en la interfaz.

Geocodificación

  • Leaflet Control Geocoder: Plugin para búsqueda y geocodificación de lugares.

Patrones de Relleno

  • Leaflet.pattern: Plugin para crear patrones de relleno personalizados en polígonos.

Estructura del Proyecto

Archivos Principales

  • indexP.js: Punto de entrada principal, inicializa la aplicación.
  • mapaP.js: Configuración y manejo del mapa base.
  • calcosP.js: Gestión de calcos (capas de dibujo).
  • simbolosP.js: Manejo de símbolos militares.
  • herramientasP.js: Herramientas de medición y análisis.
  • dibujosMCCP.js: Funciones para dibujar elementos MCC (Medidas de Coordinación y Control).
  • edicioncompleto.js: Funciones de edición para elementos en el mapa.
  • atajosP.js: Configuración de atajos de teclado.

Estilos

  • planeamiento.css: Estilos principales de la aplicación.

HTML

  • planeamiento.html: Estructura principal de la interfaz de usuario.

Funcionalidades Clave

  1. Sistema de Calcos: Permite crear, guardar y cargar calcos como capas independientes.
  2. Simbología Militar: Implementa símbolos militares estándar con opciones de personalización.
  3. Herramientas de Medición: Incluye medición de distancias y generación de perfiles de elevación.
  4. Sistemas de Coordenadas: Soporta múltiples sistemas de coordenadas y proyecciones.
  5. Búsqueda de Lugares: Integra funcionalidad de geocodificación para búsqueda de ubicaciones.
  6. Edición de Elementos: Herramientas para crear, modificar y eliminar elementos en el mapa.
  7. Atajos de Teclado: Accesos rápidos para funciones comunes.

Configuración y Personalización

La configuración del sistema se realiza principalmente a través de los archivos JavaScript. Para personalizar aspectos específicos:

  • Añadir nuevas capas de mapa: Modificar el archivo mapaP.js para incluir nuevos proveedores de mapas.
  • Personalizar símbolos: Ajustar la configuración en simbolosP.js para modificar o añadir nuevos símbolos militares.
  • Configurar sistemas de coordenadas: Editar las definiciones en mapaP.js para añadir o modificar sistemas de coordenadas.

Rendimiento y Optimización

Para mantener un rendimiento óptimo:

  • Limitar el número de elementos en un solo calco para evitar sobrecarga del navegador.
  • Utilizar la carga diferida de símbolos para mejorar el tiempo de carga inicial.
  • Optimizar las consultas de elevación para reducir el tráfico de red.

Seguridad

Consideraciones de seguridad importantes:

  • Los datos de los calcos se almacenan localmente en el navegador del usuario.
  • No se envía información sensible a servidores externos.
  • Se recomienda usar HTTPS para proteger la comunicación con servicios de mapas y geocodificación.
